Supabase 提供两个密钥：Project URL（项目地址）和 anon key（匿名公钥）。这两个值需要填入本地的 src/config.js 文件。

🔑 打开 API 设置页

操作步骤：

// src/config.js — 填入你的真实值
const SUPABASE_CONFIG = {
  url: 'https://你的项目ID.supabase.co',  // ← 替换这里
  anonKey: '你的anon公钥...'               // ← 替换这里
};
⚠️ anon key 是公钥，可以放在前端代码里（有 RLS 保护）。但不要暴露 service_role key（管理员权限，绝对不能出现在前端）。

❓ 常见问题

Q：双击打开 HTML 文件，数据不同步？

正常现象。file:// 协议无法使用 Service Worker 和跨域 API（Supabase）。必须部署到 HTTPS 才能使用云同步。本地预览可用 npx serve . 起本地服务器。

Q：iOS Safari 不弹出安装提示？

iOS PWA 只能通过「分享 → 添加到主屏幕」安装，不会自动弹出。必须用 Safari 浏览器（Chrome/微信内置浏览器在 iOS 上不支持 PWA 安装）。

Q：Vercel 部署后修改了代码怎么更新？

如果用 GitHub 方式，git push 后 Vercel 自动触发重新部署（约 30 秒）。如果用拖拽方式，重新拖拽新版本文件夹到 Vercel 即可。

Q：config.js 里的 anon key 暴露在前端安全吗？

安全。anon key 配合 RLS（行级安全）使用，每个用户只能访问自己的数据。这是 Supabase 的标准使用方式，官方文档也建议如此。绝对不要使用 service_role key 放前端。

Q：想用自定义域名（如 fund.example.com）？

在 Vercel 项目设置 → Domains 里添加你的域名，按提示配置 DNS CNAME 记录指向 Vercel，等待 DNS 生效即可（通常 5-10 分钟）。
