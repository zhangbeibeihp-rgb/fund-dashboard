🎯 推荐方案：PWA + Supabase · 一套代码双端运行

核心思路：现有 HTML 升级为 PWA，安装到手机桌面就是"APP"，用浏览器打开就是"Web"。两端共享同一个 Supabase 后端，数据自动实时同步。

为什么 PWA 而不是原生 APP？

• 个人使用无需上架应用商店（省 $99/年 Apple 开发者 + $25 Google Play）
• 不用学 Flutter/React Native，现有 HTML 直接用
• 装到手机桌面后有图标、全屏、推送通知，体验接近原生 APP
• 数据天然互通——同一个后端，Web 和 APP 看到的是同一份数据

📦 数据库表设计（4 张表）
-- Supabase PostgreSQL 建表语句 -- 1. 持仓表 CREATE TABLE portfolios ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, user_id UUID REFERENCES auth.users NOT NULL, fund_code VARCHAR(10) NOT NULL, -- 基金代码 fund_name VARCHAR(100) NOT NULL, -- 基金名称 fund_type VARCHAR(20) NOT NULL, -- mixed/index/stock share DECIMAL(12,2), -- 持有份额 amount DECIMAL(12,2), -- 持仓金额 cost_price DECIMAL(10,4), -- 成本净值 account_name VARCHAR(50) DEFAULT '默认账号', created_at TIMESTAMPTZ DEFAULT now(), updated_at TIMESTAMPTZ DEFAULT now() ); -- 2. 交易记录表 CREATE TABLE transactions ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, user_id UUID REFERENCES auth.users NOT NULL, fund_code VARCHAR(10) NOT NULL, fund_name VARCHAR(100) NOT NULL, trade_type VARCHAR(10) NOT NULL, -- buy/suy/sell amount DECIMAL(12,2) NOT NULL, -- 交易金额 share DECIMAL(12,2), -- 交易份额 nav DECIMAL(10,4), -- 成交净值 trade_date DATE NOT NULL, note TEXT, created_at TIMESTAMPTZ DEFAULT now() ); -- 3. 投资目标表 CREATE TABLE goals ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, user_id UUID REFERENCES auth.users NOT NULL, goal_type VARCHAR(30) NOT NULL, -- retirement/education/housing... goal_name VARCHAR(100) NOT NULL, target_amount DECIMAL(14,2) NOT NULL, -- 目标金额(元) monthly_invest DECIMAL(12,2) DEFAULT 3000, annual_rate DECIMAL(5,2) DEFAULT 6.0, -- 预期年化 end_age INT DEFAULT 60, is_active BOOLEAN DEFAULT true, created_at TIMESTAMPTZ DEFAULT now(), updated_at TIMESTAMPTZ DEFAULT now() ); -- 4. 风险偏好表（单行） CREATE TABLE risk_profile ( user_id UUID REFERENCES auth.users PRIMARY KEY, risk_level VARCHAR(20) DEFAULT '稳健型', -- 保守/稳健/平衡/进取/激进 invest_style VARCHAR(20) DEFAULT 'moderate', updated_at TIMESTAMPTZ DEFAULT now() ); -- RLS 策略：每个人只能看自己的数据 ALTER TABLE portfolios ENABLE ROW LEVEL SECURITY; CREATE POLICY "own_data" ON portfolios FOR ALL USING (user_id = auth.uid()); -- 其余表同理
