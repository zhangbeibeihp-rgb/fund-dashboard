## 2. 本轮已完成的 1-5 项

1. 报告更新：本文件已替换旧结论，当前验证命令不再引用已移除的 OCR API 端点。

2. 隐私、协议、数据删除：`login.html` 已增加用户协议/隐私说明勾选与弹窗；仪表盘已增加隐私说明按钮和删除数据按钮。删除动作会清理本地 `fund_goals_v*`、`fund_trades`，登录状态下会请求删除当前用户云端业务数据。

3. 投资建议表达降级：标题、描述、目标入口、按钮、目标输出、基金观察池、AI 预测信号已改为「智能测算」「目标测算」「配置复核」「偏离较高/较低」等信息工具口径。

4. 同步状态条：页面顶部已增加 `syncStatusBadge`，同步模块会显示本地保存、同步中、云端已同步、同步失败等状态。

5. OCR 与同步模块拆分：新增 `src/ocr-client.js` 和 `src/sync.js`。主 HTML 保留 UI 编排、确认弹窗和本地状态更新，识别与云端写入逻辑已拆出。

## 3. 已完成的 P0/P1 状态

### P0-1 OCR AI Key 硬编码 — 已完成

当前默认 OCR 方案为浏览器本地 Tesseract，Vercel 不需要部署 OCR AI Key，仓库也不保留 OCR AI Key 接入逻辑。`.env.example` 只保留 Supabase 配置占位，`scripts/check-secrets.mjs` 用于提交前扫描真实密钥。

### P0-2 隐私告知与同意缺失 — 已完成基础版

已补登录/注册协议勾选、仪表盘隐私说明、OCR 本地识别说明、数据删除入口。公开发布前仍建议把弹窗内容扩展为正式隐私政策和用户协议页面。

### P0-3 投资建议表达风险偏高 — 已完成基础版

用户可见的强投顾表达已降级为记录、测算、复核、偏离提示。不再把系统输出包装成买卖指令。

### P0-4 云端数据同步口径不完整 — 已完成主链路

手动交易、新增账号、OCR 导入持仓/交易、JSON 导入后的账号/持仓/交易/目标同步已统一进入 `src/sync.js`。剩余增强项是失败重试、离线队列和冲突解决。

### P0-5 生产可观测性缺失 — 已完成基础版

已增加 `/api/health`、`/api/client-log`、前端错误监听和脱敏错误上报。公开发布前仍需接入 Sentry 或等效告警。

### P1-1 PWA 缓存图标路径错误 — 已完成

`sw.js` 已改为缓存实际存在的 SVG 图标，并升级缓存版本。

### P1-2 主 HTML 过重 — 部分完成

本轮已拆出 OCR 与同步模块。后续还应拆 `charts.js`、`state.js`、`renderers.js`。

### P1-3 视觉与信息架构 — 未完成

首屏仍承载大量信息，下一阶段应收敛为资产概览、风险复核、同步状态和最近更新。

### P1-4 OCR 导入缺少确认页 — 已完成

OCR 识别后先展示确认表，用户可修改基金名、代码、金额、份额和交易记录，点击确认后才写入本地与云端。

### P1-5 测试覆盖不足 — 部分完成

已增加 OCR parser、无 Vercel OCR Key、质量检查、合规检查、模块拆分检查。后续应补浏览器 smoke test 和数据层字段映射测试。

## 4. 仍需做的后续整改

1. 补正式隐私政策和用户协议独立页面，明确数据种类、处理目的、保存期限、删除流程、第三方服务、跨境风险。

2. 增加同步失败重试和离线队列，避免弱网下用户误以为云端已保存。

3. 接入生产级错误告警，例如 Sentry、Vercel Log Drains 或同等级方案，并定义告警阈值。

4. 拆分主 HTML 的图表、渲染器和状态管理，降低后续功能迭代风险。

5. 做一次 XSS/innerHTML 安全扫描。当前页面仍大量使用 `innerHTML` 渲染，虽然部分内容经过 `escapeHtml()`，但公开发布前需要系统审查。

6. 增加 Playwright smoke test：登录页协议勾选、仪表盘渲染、导航切换、OCR 确认导入、数据删除弹窗、PWA 注册。

7. 完善注册邮件链路和国内网络访问预案。Supabase Auth、CDN 和第三方行情接口在部分网络环境下可能不稳定。

8. 做发布与回滚流程：环境变量检查、测试命令、手动验收清单、回滚入口、备份恢复步骤。
